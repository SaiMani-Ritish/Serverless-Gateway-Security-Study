# Serverless-Gateway-Security-Study
This research analyzes systemic security risks introduced by shared code and third-party libraries within serverless API gateway environments. The study employs the STRIDE threat modeling framework. It evaluates Secure Development Lifecycle (SDL) activities—ranging from A1 security training to A5 continuous monitoring—to demonstrate how "Security at the Source" principles can mitigate multi-stage remote code execution (RCE) and other software supply chain threats. By comparing serverful and serverless security models, the work recommends proactive governance, automated dependency scanning, and architectural hardening to strengthen the integrity and resilience of modern cloud-native microservices.
