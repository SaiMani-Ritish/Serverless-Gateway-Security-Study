# Serverless-Gateway-Security-Study
This research examines the systemic security risks associated with shared code and third-party libraries in serverless API gateway environments. The study utilizes the STRIDE threat modeling framework and evaluates Secure Development Lifecycle (SDL) activities, ranging from security training (A1) to continuous monitoring (A5). It demonstrates how principles of "Security at the Source" can help mitigate multi-stage remote code execution (RCE) and other threats related to software supply chains. By comparing server-based and serverless security models, the study recommends proactive governance, automated dependency scanning, and architectural hardening to enhance the integrity and resilience of modern cloud-native microservices.
